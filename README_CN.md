> **彻底解决AI上下文负债问题。**

**Prometheus（普罗米修斯）** 是全球首个通用AI上下文操作系统（ContextOS）——一个面向生产环境的平台，为任何AI系统带来**结构化上下文治理、可组合技能生态、持久化多模态记忆、知识图谱智能和多Agent编排**能力。

---

## 为什么是Prometheus？

### 问题：AI上下文负债

2026年，57%的企业在生产环境中运行AI Agent，但**95%的GenAI试点未产生可衡量的利润影响**（MIT NANDA研究，覆盖300+部署）。根源是什么？**上下文负债**——Agent推断的内容与企业真实意图之间的鸿沟。

| 失败模式 | 症状 | 根因 |
|---|---|---|
| 回答不一致 | 同一问题，不同答案 | 缺乏受控的上下文层 |
| 权威幻觉 | 自信地给出错误答案 | 更强的模型放大了坏上下文 |
| 开发通过/生产崩溃 | 测试通过，生产失败 | 环境间上下文不同 |
| 无法规模化 | 一个用例可行，多个失败 | 上下文不可组合 |
| 信任崩塌 | 没人相信输出 | 缺乏可追溯性和治理 |

> **"上下文工程已取代提示工程，成为AI编码团队的关键学科。"** — Mike Mason, ThoughtWorks (2026年1月)

### 解决方案：ContextOS

Prometheus提供完整的**上下文操作系统**：

1. **治理**——上下文的创建、验证、版本控制和注入
2. **编排**——作为可组合、可共享的Markdown驱动模块的技能
3. **持久化**——通过图+向量双引擎记忆跨会话保存知识
4. **压缩**——智能Token压缩（60-95%），保持语义不变
5. **协调**——共享上下文可见性的多Agent工作流

## 快速开始

```bash
# 安装
pip install prometheus-contextos

# 初始化上下文工作区
ptm init my-project

# 创建技能
ptm skill create "code-reviewer" --description "专业代码审查与最佳实践"

# 注册知识
ptm context add ./docs/ --recursive

# 上下文感知查询
ptm ask "这个代码库有哪些安全漏洞？"

# 运行多Agent工作流
ptm orchestrate --plan "audit_and_fix.yaml"
```

## 核心能力

### 1. 技能生态
Markdown驱动的可组合技能。像搭积木一样创建、共享和组合AI能力。

### 2. 上下文治理
带版本控制、验证和可追溯来源的结构化上下文。

### 3. 记忆层
双引擎（向量+图）持久化记忆。跨会话记忆，跨概念推理。

### 4. 知识图谱
实体-关系抽取与推理。连接整个知识库中的所有点。

### 5. 多Agent编排
共享上下文的专业化Agent协调。告别孤立的Agent孤岛。

### 6. Token压缩
60-95%的Token缩减，同时保持语义。节省成本，保持质量。
